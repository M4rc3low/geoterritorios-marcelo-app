# Screenshots

This document will store the main visual references for the project.

## Required screenshots

Add screenshots after the application is running locally or deployed.

Recommended captures:

1. Territory dashboard overview
2. Map view
3. Building and apartment control
4. Coverage indicators
5. Mobile responsive layout

## Recommended folder structure

```txt
public/
└── screenshots/
    ├── dashboard.png
    ├── map.png
    ├── buildings.png
    ├── coverage.png
    └── mobile.png
```

## README integration

After adding screenshots, include them in `README.md` using:

```md
## Screenshots

![Dashboard](public/screenshots/dashboard.png)
![Map](public/screenshots/map.png)
```

## Quality standard

- Use real application screens
- Avoid personal or sensitive address data
- Prefer clean sample data
- Capture desktop and mobile views
- Keep images optimized for GitHub display
